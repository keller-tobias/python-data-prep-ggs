# Git Tutorial Guide

Welcome to this comprehensive Git tutorial series! This guide will walk you through essential Git operations using both command line and **Visual Studio Code (VS Code)**.

## Prerequisites

Before starting this tutorial, ensure you have:
- Git installed on your system
- VS Code installed
- A GitHub account (for remote repository exercises)
- Basic familiarity with command line operations

Please refer to the main [README.md](../README.md) file for detailed setup instructions.

## Tutorial Structure

This tutorial is designed to build your Git knowledge progressively. Each section is self-contained with consistent numbering starting from 1:

### 1. [Basic Git Workflow](01-basic-git-workflow.md)
**Topics covered:**
- Basic Git workflow: initialization, tracking changes, undoing changes (Section 1)
- Branching: creating, merging, and resolving conflicts (Section 2)
- Managing files with .gitignore (Section 3)

**What you'll learn:** The fundamental Git workflow, branching strategies, and how to manage your local repository.

### 2. [Collaboration with Remote Repositories](02-collaboration.md)
**Topics covered:**
- Working with GitHub: creating repositories and connecting to remotes (Section 1)
- Synchronizing changes: pushing and pulling (Section 2)
- Understanding fetch vs pull (Section 3)
- Topics out of scope for this workshop (Section 4)

**What you'll learn:** How to work with remote repositories and collaborate using GitHub.

### 3. [Using Git in VS Code](03-git-in-vscode.md)
**Topics covered:**
- Source Control panel and file status icons (Section 1)
- Basic Git operations: staging, committing, and viewing history (Section 2)
- Branch management in VS Code (Section 3)
- Remote operations (Section 4)

**What you'll learn:** How to leverage VS Code's built-in Git integration for an efficient workflow.

### 4. [Resources and Best Practices](04-resources.md)
**Topics covered:**
- Git best practices for commits and branching (Section 1)
- Essential Git commands reference (Section 2)
- Common Git scenarios and solutions (Section 3)
- Learning resources (Section 4)
- Key terms glossary (Section 5)
- Getting help (Section 6)

**What you'll learn:** Professional Git practices and where to go for further learning.

## How to Use This Tutorial

### Learning Approach
- **Sequential**: Work through sections 1-4 in order for best results
- **Tutorial + Exercise**: Each section includes guided tutorials followed by hands-on exercises
- **Practice-focused**: Emphasis on actually using Git, not just reading about it

### Tutorial Format
Each section follows a consistent structure:
- 📖 **Tutorials**: Step-by-step guided instructions
- 🏋️ **Exercises**: Hands-on practice to reinforce learning
- 💡 **Tips**: Helpful insights and best practices
- ⚠️ **Warnings**: Important notes about potential pitfalls

### Estimated Time
- **Total time**: 3-4 hours for complete tutorial
- **Section 1**: 60-90 minutes (fundamental concepts)
- **Section 2**: 45-60 minutes (remote repositories)
- **Section 3**: 60-75 minutes (VS Code integration)
- **Section 4**: 30 minutes (reference and best practices)

## Important Notes

> 💡 **Practice Repository**: Consider creating a dedicated practice repository for these exercises so you can experiment freely without affecting important work.

> ⚠️ **Command Differences**: While this tutorial focuses on VS Code, all operations can also be performed via command line. Both approaches are shown where relevant.

> 🎯 **Workshop Focus**: This tutorial covers the most essential Git concepts needed for collaborative development. For advanced topics like rebasing, hooks, or complex merge strategies, see the resources section.

## Getting Help

If you encounter issues:

1. **Check the troubleshooting section** in [Resources](04-resources.md)
2. **Use Git's built-in help**: `git help <command>`
3. **Refer to the glossary** for term definitions
4. **Practice in a test repository** if you're unsure about a command
